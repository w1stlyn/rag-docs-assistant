# Технология RAG: краткий обзор

Retrieval-Augmented Generation (RAG) — гибридная архитектура, объединяющая
семантический поиск по векторным представлениям документов и генерацию ответа
большой языковой моделью.

## Компоненты

1. **Loader** — загружает документы из PDF, DOCX, Markdown.
2. **Splitter** — режет текст на фрагменты по 400–800 токенов с перекрытием.
3. **Embedder** — превращает фрагменты в векторы фиксированной размерности.
4. **Vector store** — хранит векторы и выполняет поиск ближайших соседей.
5. **LLM** — генерирует ответ на основе исходного вопроса и найденного контекста.

## Преимущества

- Знания можно обновлять без переобучения модели.
- Ответы привязаны к источнику, что снижает риск галлюцинаций.
- Конфиденциальные данные не покидают инфраструктуру владельца.

## Метрики качества

Для оценки RAG-систем применяются метрики проекта RAGAS:
Context Precision, Context Recall, Faithfulness и Answer Relevancy.
